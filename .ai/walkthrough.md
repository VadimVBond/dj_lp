# Отчет об изменениях: Исправление переключателя языков

## Что было сделано
1. **Обновление `templates/includes/header.html`**:
   - Изменен способ генерации ссылок для переключения языков.
   - Вместо жестко заданного `/{{ lang_code }}/` теперь используется `{{ request.META.SCRIPT_NAME|default:"/" }}{{ lang_code }}/`.
   - **Добавлена ссылка на логотип**: теперь логотип ведет на главную страницу текущего языка с учетом `SCRIPT_NAME`.
   - Это гарантирует, что при деплое на GitHub Pages все ссылки будут вести на `/dj_lp/en/` (или другие языки), а не на корень домена.

## Почему это важно для GitHub Pages
GitHub Pages размещает проекты в подпапках (например, `/dj_lp/`). Стандартные абсолютные ссылки Django (начинающиеся с `/`) ведут в корень домена (`username.github.io/`), что вызывает ошибки 404. Использование `SCRIPT_NAME` позволяет динамически подставлять базовый путь проекта.

## Результат
- Локально ссылки работают как `/en/`, `/ru/` и т.д.
- При экспорте через `django-distill` с настройкой `FORCE_SCRIPT_NAME = "/dj_lp/"` ссылки превратятся в `/dj_lp/en/`, `/dj_lp/ru/`, что соответствует структуре GitHub Pages.
